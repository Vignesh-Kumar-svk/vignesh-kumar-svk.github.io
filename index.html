// how it's built

Three systems, one pattern

Replace synchronous, manual steps with async, idempotent, self-healing pipelines.

01

Decouple with messaging

RabbitMQ pulls service communication off the synchronous path — OpenStack calls, HAProxy config, and capacity polling all run in parallel instead of blocking each other.

02

Pool, don't reconnect

Persistent VDOM sessions and multiplexed SSH channels turn the Fortinet engine's per-request connection overhead into a one-time cost — the source of the 15s → 4ms jump.

03

Design for failure, not just success

Checkpoint-based state recovery and idempotent retries mean the provisioning workflow heals itself on transient failure instead of needing a human to re-run it.
